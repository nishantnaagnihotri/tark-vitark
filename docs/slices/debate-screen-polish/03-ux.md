## UX Flows

### Item 1 — Mobile Card Width

**Mobile (≤480 px)**

| Step | Description |
|---|---|
| 1 | Visitor opens the debate screen on a mobile device |
| 2 | Topic heading visible, full-width |
| 3 | Legend bar visible below topic |
| 4 | Argument cards display as chat-style messages. Each card now occupies 85% of the viewport width (up from ~72%) — wider text lines improve readability |
| 5 | Tark cards (blue) are left-aligned; Vitark cards (amber) are right-aligned with a 15% inset from the left. Stagger creates conversational back-and-forth visual rhythm |
| 6 | Bubble tails remain correctly anchored: Tark tail at card left edge, Vitark tail at card right edge |
| 7 | Visitor scrolls vertically to read the debate thread |

No flow change for tablet/desktop (card width change applies only to ≤480 px).

### Item 2 — Spine Dot Row Fix

**Tablet/Desktop (≥481 px)**

| Step | Description |
|---|---|
| 1 | Visitor opens debate screen at ≥768 px viewport |
| 2 | Timeline renders with a vertical center spine |
| 3 | Each argument card has a small circular dot on the spine — the dot should align vertically with the top of the card's text content (row 1 of the card's grid substructure) |
| 4 | **Before fix:** Vitark spine dots render at row 2 (one row below the Tark dots), causing visual misalignment |
| 5 | **After fix:** all spine dots (Tark and Vitark) render at row 1 — vertically aligned to the card top for a clean, consistent timeline |
