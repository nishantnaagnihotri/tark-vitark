## 3. UX Flow Inventory

### Flow A — Podium Entry (all tiers, same interaction model)

```
Resting state (FAB visible, collapsed)
  → User taps FAB
  → FAB expands: tark and vitark mini-buttons appear
  → User taps tark or vitark mini-button
  → [FAB collapses] + [Scrim fades in] + [BottomSheet rises from bottom]
  → User edits text in textarea
  → User taps "Post" / publish button
  → [BottomSheet closes] + [Scrim fades out] + [Post appears in Timeline]
  → Return to: Resting state (FAB visible, collapsed)
```

Dismiss branch:
```
  → User taps scrim OR drag-handles sheet below 80 px threshold
  → [BottomSheet closes] + [Scrim fades out] + [No post published]
  → Return to: Resting state (FAB visible, collapsed)
```

### Flow B — Theme Toggle (passive)

No interaction required; theme change while sheet is open must immediately update scrim opacity via CSS variable cascade. Dev CSS must support:
- `[data-theme="light"]` at ≥768 px: scrim `rgba(0,0,0,0.32)` tablet / `rgba(0,0,0,0.36)` desktop
- `[data-theme="dark"]` at ≥768 px: scrim `rgba(0,0,0,0.48)` tablet / `rgba(0,0,0,0.52)` desktop

  @media (min-width: 768px) and (max-width: 1023px) {
