# UX Figma Agentic Protocol

Purpose: enable UX Strategist to produce Figma artifacts for pending slices with minimum intervention and maximum design-to-code reuse.

## Agent-First Rule
- UX artifact production is agent-first by default.
- Manual design operations are fallback-only when Figma API/MCP access is unavailable.
- Fallback output must still be generated by agent in structured form so human action is reduced to copy/paste and link placement.

## One-Time Setup (Per Repository)
1. Create one shared Figma file for product slices.
2. Create pages by domain (for example: Debate, Onboarding, Profile).
3. Establish reusable styles/variables:
   - color tokens
   - typography scale
   - spacing scale
   - radius/shadow tokens
4. Create base reusable components:
   - page container
   - section heading
   - content card
   - status/empty/error block
5. Store Figma file link in `docs/current-state.md` once available.

## Slice Input Contract
Before UX Gate execution, UX Strategist loads:
1. `AGENTS.md`
2. `docs/current-state.md`
3. `docs/non-functional-requirements-baseline.md`
4. `docs/functional-requirements-index.md`
5. active `docs/functional-requirements-*.md`
6. active `docs/ux-spec-*.md`
7. `docs/decision-log.md`

## Deterministic UX Outputs (Per Slice)
UX Strategist must produce all of these outputs:

1. UX Spec update
- Fully populated `docs/ux-spec-<slice>.md`.

2. Figma production contract block inside UX spec
- Page name
- Frame names for mobile/tablet/desktop variants
- Component instance list
- Token usage list
- State matrix (default/empty/loading/error/success as applicable)

3. Code translation contract block inside UX spec
- `figma_frame -> frontend_component` mapping
- `figma_component -> reusable_ui_component` mapping
- `token_name -> code_variable` mapping
- `interaction/state -> implementation expectation` mapping

4. Figma reference
- Final frame URL in UX spec and linked issue.
- If API unavailable, include fallback marker: `Pending-manual-link` and open question in current-state.

## Naming Convention (Reuse + Translation)
- Frames: `<slice>-<viewport>-<state>`
  - Example: `debate-screen-mobile-default`
- Components: `cmp/<domain>/<name>/<variant>`
  - Example: `cmp/debate/argument-card/favour`
- Tokens: `tk/<category>/<semantic>/<scale>`
  - Example: `tk/color/surface/favour-bg`

## Fallback Mode (No Figma API/MCP)
When direct Figma automation is blocked:
1. UX Strategist still completes UX spec including production and translation contracts.
2. UX Strategist generates a precise frame/component checklist for manual creation.
3. Human performs minimal action: create/update frames and paste URLs.
4. UX Strategist immediately syncs links into repo docs and issue.
5. UX Strategist logs blocker and next automation step in `docs/current-state.md` and `docs/decision-log.md`.

## Done Criteria
- UX spec is complete and implementation-ready.
- Figma link is present (or temporary fallback marker is explicitly tracked).
- Code translation contract is complete and unambiguous.
- `docs/current-state.md` reflects gate progress and open blockers.
- Linked issue references UX spec and Figma artifact.
