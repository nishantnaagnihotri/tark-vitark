Foundation

Core Principles

🎯
One slice at a time. WIP limit of 1. Every slice goes through the gate pipeline before the next begins.
🧑‍💼
Product Owner owns all decisions. Agents prepare; humans approve scope, design, and PRs. Merges are always PO-only.
🔒
Gates are not optional. No gate can be skipped (except Trivial complexity, which bypasses Gates 2–4).
↩️
Loop-back is the norm. Every gate has a feedback path. Agents iterate until the gate passes.

8 / 8
Rules

Key Constraints

🚫
Agents never merge. Product Owner personally reviews and merges pull requests.
🏠
Gate 3 is local-only. UX + Design work and Design QA always run in the local environment — no cloud execution.
✅
Copilot review exit criterion: the latest review body must say "generated 0 comments". Thread-level analysis alone is not sufficient.
🔁
Trivial slices skip Gates 2, 3, and 4 — but Gate 5.5 Runtime QA still applies if the change is UI-impacting.
